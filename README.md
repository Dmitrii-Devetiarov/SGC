# 🧪 SGC — Scientific Gap Cartography

Интерпретируемый движок для генерации научных гипотез на основе анализа структурных разрывов в базе знаний.

## 💡 Как это работает

SGC не имитирует мышление эксперта. Вместо этого он видит то, что человеку недоступно — **структурные разрывы в многомерном пространстве знаний**.

1. **Векторное облако** — все документы (статьи, отчёты, патенты) превращаются в эмбеддинги и образуют облако точек в латентном пространстве.
2. **Поиск антикластеров** — UMAP снижает размерность, HDBSCAN находит кластеры знаний. Пустоты между кластерами — это **войды**, потенциальные научные пробелы.
3. **Вербализация** — граничные документы вокруг каждого войда подаются в LLM, которая формулирует проверяемую гипотезу строго на основе этих источников.

**Каждая гипотеза трассируется до конкретных абзацев исходных документов.** Никаких галлюцинаций.
## 🚀 Быстрый старт

### Предварительные требования

- Python 3.10+
- [YandexAI API ключ](https://console.yandex.cloud/) (предоставлен организаторами)
- ChromaDB CLI (для WSL/Windows)

### Установка

```bash
git clone <repo-url>
cd SGC
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Конфигурация

1. Скопируйте `.env.example` в `.env` и заполните ключи:
   ```bash
   cp .env.example .env
   # Отредактируйте .env: YANDEX_API_KEY, YANDEX_FOLDER_ID
   ```

2. (Опционально) Настройте `config.yaml` под свои данные.

### Демо-база

Если у вас нет своих данных, распакуйте демо-базу (учебники по металлургии и обогащению):

```bash
tar -xzf db_demo.tar.gz
```

### Запуск

**Терминал 1 — ChromaDB сервер:**
```bash
chroma run --path ./db --port 8000
```

**Терминал 2 — Приложение:**
```bash
streamlit run app.py
```

Откройте http://localhost:8501 в браузере.

   python src/indexer.py
   ```

## 📁 Архитектура проекта

### Основные компоненты

**app.py** — Streamlit-интерфейс. Точка входа для пользователя. Боковая панель с настройками, поле запроса, визуализация карты знаний, экспорт гипотез в DOCX/JSON.

**src/parser.py** — Парсер документов. Извлекает текст из PDF (цифровых и сканов через OCR), TXT, CSV, Excel. Для сканированных PDF определяет наличие изображений и при необходимости готовит страницы к распознаванию.

**src/chunker.py** — Разбивка текста на чанки. Настраиваемый размер (по умолчанию 512 токенов) с перекрытием. Фильтрация пустых и бессмысленных фрагментов. Кеширование OCR-результатов через ChromaDB.

**src/yandex_api.py** — Враппер для YandexAI API. Эмбеддинги (text-embeddings-v2-doc/query), генерация текста (YandexGPT Pro 5.1, Lite 5), OCR сканов (Qwen3.6 35B), перевод, извлечение предикатов.

**src/vectordb.py** — Векторное хранилище на ChromaDB. Сохранение чанков с метаданными, проверка кеша OCR, выгрузка для анализа, поиск ближайших соседей.

**src/gap_finder.py** — Ядро системы. UMAP-проекция, HDBSCAN-кластеризация, геометрический поиск войдов (антикластеров), фильтрация перекрытий, сортировка по глубине.

**src/verbaizer.py** — RAG-вербализация гипотез. Превращает граничные чанки войда в структурированную гипотезу через YandexGPT Pro 5.1 с жёсткими ограничениями (только на основе источников, без выдумывания).

**src/indexer.py** — Индексация документов. Сквозной пайплайн: парсинг, чанкинг, эмбеддинги, сохранение в ChromaDB. Поддерживает инкрементальную запись.

**src/exporter.py** — Экспорт гипотез в DOCX и JSON.

**src/utils.py** — Вспомогательные функции: загрузка конфига с подстановкой .env, логирование, косинусное сходство.

### Данные

**data/input/** — Исходные документы для индексации.

**db/** — Персистентное хранилище ChromaDB (создаётся после индексации или распаковки демо-базы).

**output/** — Результаты экспорта гипотез.

### Конфигурация

**.env** — API-ключи (не коммитится).

**config.yaml** — Параметры: размеры чанков, UMAP, HDBSCAN, поиск войдов, ранжирование.

### Тесты

**tests/test_ocr_e2e.py** — Сквозной тест OCR-пайплайна.

**tests/test_umap_viz.py** — Визуализация UMAP-проекции.

**tests/test_autogap.py** — Автоматический поиск войдов и вербализация.

**tests/test_verbaizer.py** — Поиск по запросу с вербализацией.

## 🎯 Примеры запросов

Система работает в двух режимах:

**По запросу** — ищет пробелы, релевантные конкретной цели. Точка запроса отображается на карте красной звездой, показывая близость к существующим областям знаний.

**Автогенерация** — находит все структурные разрывы в базе и ранжирует по глубине.

### Примеры запросов для демо-базы (металлургия, обогащение)

- повысить производительность флотации золота на 15 процентов
- снизить расход реагентов при флотации медных руд
- увеличить извлечение серебра из упорных руд
- уменьшить энергопотребление при измельчении
- подобрать реагент для разделения сульфидных минералов

### Формат гипотезы на выходе

Для каждой гипотезы система выдаёт:

- Гипотеза — одно предложение, что именно предлагается проверить
- Механизм — как это должно работать физически или химически
- Обоснование — какие фрагменты источников указывают на гипотезу
- Новизна — почему это не описано в известных источниках
- Риски — технические и экономические риски проверки
- План проверки — 2-3 шага с критериями успеха и провала
- Источники — список документов с номерами страниц

## 🔍 Интерпретируемость

Каждая гипотеза SGC полностью трассируется до исходных документов. Никаких галлюцинаций и непрозрачных выводов.

### Как это достигается

**Геометрическое ядро, а не вероятностный вывод**

Гипотеза рождается не из вероятностного распределения токенов, а из структурного разрыва в пространстве эмбеддингов. Войд — это математически доказуемая пустота между кластерами знаний. Эксперт может увидеть его на UMAP-карте: красные круги на левом графике, голубые пунктирные круги на тепловой карте.

**RAG строго на граничных документах**

LLM используется только для вербализации — перевода математической аномалии в читаемый текст. При этом модель получает исключительно граничные чанки (до 10 фрагментов) и жёсткий промпт: формулировать гипотезу только на их основе, не добавляя новых фактов. Если данных недостаточно — модель честно сообщает об этом.

**Трассировка до абзаца**

Каждая гипотеза сопровождается списком источников с номерами страниц. Эксперт может кликнуть по источнику и увидеть конкретный фрагмент текста, который послужил основанием для вывода. Цепочка от войда до формулировки полностью прозрачна.

**Human-in-the-Loop**

Эксперт управляет поиском через понятные параметры: глубина разрыва (от осторожных гипотез до фронтирных), дробность тем (количество кластеров). Любую гипотезу можно проверить, отклонить или использовать как основу для дальнейшего исследования.


## ⚠️ Ограничения MVP и Roadmap

### Что реализовано в MVP

- Парсинг PDF (цифровых и сканов через OCR), TXT, CSV, Excel
- Перевод всех документов на русский язык (устранение межъязыковой дистанции)
- Поиск научных пробелов геометрическим методом (антикластеры)
- Ранжирование гипотез по глубине разрыва
- RAG-вербализация гипотез строго на граничных документах
- Визуализация карты знаний (UMAP + тепловая карта плотности)
- Экспорт гипотез в DOCX и JSON
- Human-in-the-Loop: настройка глубины разрыва и дробности тем
- Полная трассировка гипотез до исходных абзацев
- Детектор противоречий в граничных документах (базовый)
- Локальное развёртывание ядра, безопасность через необратимость эмбеддингов

### Ограничения MVP

- Нет контрфактуального анализа (что если параметр был бы другим)
- Нет предсказательных моделей (экстраполяция известных данных)
- KPI-ранжирование по новизне, реализуемости, эффекту не реализовано — только по глубине разрыва
- Китайский язык поддерживается через машинный перевод, не нативно
- Нет визуального конструктора дорожных карт проверки гипотез
- Нет интеграции с Jira/YouTrack через REST API
- OCR сканов требует YandexAI API — не работает полностью офлайн

### Roadmap

**v1.5 — Быстрые улучшения (2 недели)**

- Контрфактуальный анализ: поиск параметров с узким диапазоном значений и экстраполяция
- KPI-скоринг гипотез по новизне, реализуемости, ожидаемому эффекту и рискам
- Визуальный конструктор дорожных карт проверки гипотез
- Локальные LLM (Llama 3) для конфиденциальных внедрений
- Детектор информативности чанков: отсев методологического мусора

**v2.0 — Глубина анализа (2 месяца)**

- Тонкая семантическая разметка слоёв: контекст, методология, дискуссия
- Учёт графа цитирований как гравитации в UMAP-пространстве
- Каузальные графы влияния: извлечение причинных связей из текста
- Коннекторы к Jira/YouTrack через REST API с настраиваемым маппингом полей
- Полноценный NER для материаловедения (Fine-tuned SciSpacy)

**v2.5 — Предиктивный KPI (3 месяца)**

- ML-скоринг рисков и затрат на исторических данных R&D проектов
- Активное обучение на обратной связи: система учитывает подтверждённые и опровергнутые гипотезы
- Байесовская оценка неопределённости гипотез

**v3.0 — Мультимодальность (6+ месяцев)**

- Поиск войдов в эмбеддингах графиков, таблиц, спектрограмм
- Кросс-модальные гипотезы (текст плюс изображение плюс структурированные данные)
- Автоматическое пополнение базы из открытых источников (Materials Project, Semantic Scholar)
- Нативная поддержка китайского и японского языков в эмбеддингах
